# 流程總覽

## AS-IS — 現況（人工關鍵字搜尋）

```
顧問有問題
  → 打開 SharePoint / Confluence
  → 輸入關鍵字
  → 逐一開啟文件瀏覽
  → 自行判斷哪段相關
  → 整理答案（手動摘要）
平均耗時：30 分鐘
痛點：關鍵字不精準、文件太長、結果散落多處
```

## TO-BE — 目標（AI 多 Agent 問答）

```
顧問輸入自然語言問題
  → Orchestrator 接收問題
  → Search Agent 語意搜尋 → 回傳相關段落（含來源）
  → Summarize Agent 壓縮段落為結構化摘要
  → Write Agent 產出最終回答（含引用）
  → 顧問收到答案
目標耗時：< 30 秒
改善：不需知道關鍵字、自動整合多份文件、答案有引用可追溯
```

## 流程差異分析

| 步驟 | AS-IS | TO-BE | 改善幅度 |
|------|-------|-------|----------|
| 搜尋方式 | 關鍵字 | 語意向量 | 精準度大幅提升 |
| 結果整合 | 人工閱讀多文件 | 自動摘要 | 時間從 30 分降至 < 30 秒 |
| 來源追溯 | 需自行記錄 | 自動附引用 | 可信度提升 |
| 知識庫涵蓋 | 依賴搜尋技巧 | 語意覆蓋全庫 | 漏查機率降低 |
