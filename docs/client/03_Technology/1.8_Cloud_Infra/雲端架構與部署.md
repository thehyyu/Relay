# 雲端架構與部署

## 部署環境一覽

| 版本 | 環境 | 基礎設施 | LLM | 網路邊界 |
|------|------|----------|-----|----------|
| v1 | 開發者本機 | 單一 Python script | Ollama（本機） | 完全本機 |
| v2a | 開發者本機 | Docker Compose | Ollama（本機） | 完全本機 |
| v2b | 開發者本機 | minikube（單節點 K8s） | Ollama（本機） | 完全本機 |
| v3 | 混合 | 容器 + Serverless Functions | Claude API（雲端） | 部分外部 |

## 各版本架構說明

### v1 — Monolith

```
[使用者 CLI]
     ↓ 直接呼叫
[Python script：Orchestrator + All Agents]
     ↓ 本地檔案
[ChromaDB（本機目錄）]
     ↓ HTTP
[Ollama（Mac mini :11434）]
```

無容器化，所有邏輯在單一 process。

---

### v2a — Microservices（Docker Compose）

```
[使用者 CLI]
     ↓ HTTP
[Orchestrator :8000]
     ↓ HTTP
[Search Agent :8001] → [ChromaDB container :8000]
[Summarize Agent :8002]
[Write Agent :8003]
     全部 → [Ollama（Mac mini :11434）]
```

各服務獨立容器，透過 Docker 內部網路溝通。

---

### v2b — Kubernetes（minikube）

```
[Ingress]
     ↓
[Orchestrator Deployment（replicas: 2）]
     ↓ Service
[Search Agent Deployment] → [ChromaDB StatefulSet + PVC]
[Summarize Agent Deployment]
[Write Agent Deployment]
     全部 → [Ollama Service（NodePort）]
```

ChromaDB 用 StatefulSet + PersistentVolumeClaim 保護資料。

---

### v3 — Hybrid Serverless

```
[使用者介面]
     ↓ HTTP
[Orchestrator（容器，長駐）]
     ↓ Message Queue
[Search Agent（Serverless Function，短任務）]
[Summarize Agent（Serverless Function，短任務）]
[Write Agent（Serverless Function，短任務）]
     ↓
[Claude API（外部，HTTPS）]
[ChromaDB（雲端儲存）]
```

Orchestrator 常駐，其他 Agent 按需啟動（冷啟動有延遲）。

## 甲方 IT 限制

- v1 ~ v2b：所有推論必須在甲方防火牆內完成
- v3：使用 Claude API 前需提交資安評估報告，IT 核准後才能上線
- 所有對外 port 需申請開放，禁止直接暴露 ChromaDB 與 Ollama
